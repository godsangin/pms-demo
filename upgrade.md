## 프로젝트 진척관리 방법론

### 프로젝트 관리단계 : 하위 테스크 등록 가능
 - 각 테스크 별 수행기관, 담당자, 진척률 정보 기재 가능하도록
 - 각 테스크 별로 산출물을 입고할 수 있도록 구성
 - 단위테스트, 통합테스트의 경우 결함을 등록하고 상태(조치예정, 조치 중, 조치완료)를 기입할 수 있도록 구성
1. 사업관리(점유비 10%)
 - 착수보고, 중간보고, 종료보고 진척률 입력(0 or 100%)
 - 주간보고, 월간보고 진척률 입력(프로젝트 전체 기간에 따른 진척률 자동산정)
2. 분석/설계(점유비 30%) : 정의된 현황정보를 csv로 내려받을 수 있도록 구성
 - 하위 테스크 총량 대비 진척률 평균 산정
 - 테일러링 결과에 따른 산출물 기준 진척률 입력
3. 개발/단위테스트(점유비 40%) : 정의된 현황정보를 csv로 내려받을 수 있도록 구성
 - 스프린트 기반의 주간진척 관리(각 하위 테스크 별 진척률 입력)
 - 하위테스크는 프로그램 목록으로 정의(관리화면에서는 수행기관 별 진척사항으로 표출)
 - 생산성 관리(수행기관, 담당자 별 진척률)
 - 단위테스트 관리(프로그램 목록 별 테스트 진척사항 입력, 관리화면에서 단위테스트 수행현황에 대한 UI 별도 정의)
4. 통합테스트(점유비 15%)
 - 통합테스트 시나리오를 하위 테스크로 입력
 - 각 시나리오 별 진척관리
5. 이행(점유비 5%)
 - 이행관련 하위테스트 등록, 진척관리 기능 구성
